# Структура проекта

## Особенности
- Монорепозиторий
- Мультимодульный проект
- Гексагональная архитектура

## Монорепозиторий

Все части проекта хранятся в одном репозитории, в соседних папках на верхнем уровне. Эти части можно разделить на несколько групп:
* информационные (документация, логи принятых решений, настройки деплоя),
* фронт,
* логика работы приложения,
* адаптеры (в терминах гексагональной архитектуры)

## Мультимодульный проект

- `adr/` - ADR
- `contract/` - Контракты взаимодействия бэк-фронт, бэк-очереди, бэк-сторонние сервера
- `docs/` - Документация по проекту
- `app/` - Первичная точка входа, настройки Spring. Содержит тесты контекста приложения.
- `postgres/` - (позже) Адаптер к Postgres. Реализует интерфейсы источников данных. Содержит @DataR2dbcTest тесты
- `kafka/` - (позже) Адаптер к Kafka. Реализует интерфейсы источников данных
- `feign/` - (позже) Адаптер к HTTP-запросам на сторонние серверы. Реализует интерфейсы источников данных.
- `web-api/` - HTTP API. Реализует контроллеры, модели и мапперы (отправляющиеся структуры по сети <---> модели, используемые в приложении), заботится об версионировании api. Обращается к интерфейсам вариантов использования. Содержит `@MockMvc тесты на моках вариантов использования, тесты контроллирут валидацию запросов, корректность кодов ответа, права доступа, соответствие openapi-контракту
