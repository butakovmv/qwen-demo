# OTUS Application

Веб-приложение для прохождения опросов/тестов. Пользователь загружает вопросы, вводит ответы и отправляет их на сервер.

## Быстрый запуск

### `make demo` — полная сборка и запуск

Одна команда соберёт все артефакты в изолированном build-контейнере и запустит стек:

```bash
make demo
```

После завершения:
- **Frontend**: http://localhost:80
- **Backend API**: http://localhost:81/api/v1
- **Actuator**: http://localhost:82/actuator/health
- **PostgreSQL**: localhost:83

### `make dev` — режим разработки

Собирает артефакты локально и запускает контейнеры в режиме **watch** — изменения автоматически синхронизируются:

```bash
make dev
```

При изменении backend-кода:
```bash
./gradlew :app:jar :app:copyDependencies && touch ./app/build/libs/app-0.0.1-SNAPSHOT.jar
```

При изменении frontend-кода:
```bash
cd front && npm run build
```

## Структура проекта

```
.
├── app/            # Spring Boot приложение (точка входа)
├── operation/      # Use-case операции (бизнес-логика)
├── web-api/        # REST-контроллеры (WebFlux)
├── postgres/       # R2DBC-репозитории
├── front/          # Vue 3 + TypeScript SPA
├── ui-test/        # Playwright smoke-тесты
├── arch-tests/     # ArchUnit архитектурные тесты
├── config/         # Конфигурации (detekt, ktlint)
└── docs/           # Документация
```

## Анализ ошибок и корректировок Qwen

Ниже — разбор случаев, когда Qwen ошибался, галлюцинировал или когда пользователь корректировал направление работы. Данные получены из анализа истории диалогов (`qwen_log.txt`).

### 1. Неправильное имя папки для фронтенда

| | |
|---|---|
| **Запрос** | «создай папку для проекта на typescript и vue3» |
| **Ошибка Qwen** | Сам придумал имя `vue3-typescript-app` |
| **Корректировка** | «создай папку **front** для проекта на typescript и vue3» |

### 2. Ручная установка Gradle вместо Wrapper

| | |
|---|---|
| **Запрос** | «установи gradle по официальной инструкции» |
| **Ошибка Qwen** | Пытался качать Gradle через `curl` — ушло в таймаут |
| **Корректировка** | Qwen сам предложил Gradle Wrapper как альтернативу |

### 3. Зацикливание на Gradle Wrapper

| | |
|---|---|
| **Запрос** | «пропусти шаг с установкой gradle wrapper, будем использовать уже установленный в системе gradle» |
| **Ошибка Qwen** | Трижды пытался создать Wrapper (таймауты), не предлагая альтернативу |
| **Корректировка** | Пользователь явно потребовал пропустить этот шаг |

### 4. Галлюцинация: несуществующий алиас зависимости

| | |
|---|---|
| **Контекст** | Настройка `usecase/build.gradle.kts` |
| **Ошибка Qwen** | Написал `implementation(kotlin("coroutines-core"))` — такого алиаса нет |
| **Исправление** | Сам обнаружил при сборке → `org.jetbrains.kotlinx:kotlinx-coroutines-core:1.9.0` |

### 5. Нарушение архитектуры: @Component в usecase-модуле

| | |
|---|---|
| **Ошибка Qwen** | Поставил `@Component` в `HelloWorldUseCaseImpl` — модуль `usecase` не должен зависеть от Spring |
| **Корректировка** | «здесь можно сделать исключение, имплементация useCase может зависеть от spring context» |

### 6. BootJar для всех модулей вместо одного

| | |
|---|---|
| **Ошибка Qwen** | Применил `org.springframework.boot` к `web-api` и `usecase` — каждый создавал свой bootJar |
| **Исправление** | Убрал плагин, оставил только в `app`, в остальных — `dependencyManagement` с BOM |

### 7. Серия ошибок в resolutionStrategy для detekt

| | |
|---|---|
| **Контекст** | Конфликт версий Kotlin между Spring DM и detekt |
| **Ошибки Qwen** | 4+ итераций: неправильная область видимости (`name == "detekt"`), случайное добавление `detektPlugins` в root, попытка использовать detekt 2.0.0-alpha.2 |
| **Исправление** | В итоге просто обновил detekt до совместимой версии |

### 8. Выдуманные имена Gradle-задач

| | |
|---|---|
| **Ошибка Qwen** | Написал `dependsOn(":front:npm", ":front:npmFormat")` — таких задач нет |
| **Исправление** | Сам обнаружил → `:front:npmFormatFix` |

### 9. Циклическая зависимость в Gradle

| | |
|---|---|
| **Ошибка Qwen** | Добавил `npmBuild.finalizedBy(tarDist)` при существующем `tarDist.dependsOn(npmBuild)` |
| **Сам Qwen** | «Стоп, это цикл» |
| **Исправление** | Убрал `finalizedBy` |

### 10. Случайное удаление тестовых файлов

| | |
|---|---|
| **Контекст** | Разделение `PostgresRepositoriesTest` |
| **Ошибка Qwen** | `rm -rf` на папку `test/` — удалил `PostgresTestConfig.kt`, `schema-h2.sql`, `application.yml` |
| **Исправление** | Воссоздал все файлы заново |

### 11. Невалидный тип в launch.json

| | |
|---|---|
| **Ошибка Qwen** | Создал `.vscode/launch.json` с `"type": "shell"` — невалидный тип |
| **Исправление** | Предложил использовать `tasks.json` вместо `launch.json` |

### 12. Переусложнение с buildSrc

| | |
|---|---|
| **Задача** | Кэширование npmInstall |
| **Ошибка Qwen** | Создал отдельный класс `NpmInstall` в `buildSrc/`, затем встроил в `build.gradle.kts` — серия ошибок компиляции (`RegularFileProperty`, лямбда `exec`) |
| **Сам Qwen** | «Стоп, это слишком сложно. Давайте проще» |

### 13. Неправильный порядок репозиториев

| | |
|---|---|
| **Ошибка Qwen** | Поставил `gitverse.ru` и `sbercloud.ru` перед `mavenCentral()` — зеркала нестабильны |
| **Исправление** | Поменял порядок — `mavenCentral()` первым |
